# QAチェックリスト生成ツール

Webディレクター向けの「QAを行うためのチェックリスト生成」Webツールです。

## 概要

条件（技術スタック / 新規・リニューアル）に応じて、公開前QAを中心にチェック項目を表示します。

- ログイン不要・保存不要
- 1ページ完結のシンプルなUI（SP対応）
- チェックリストをMarkdown形式でコピー可能
- セクション折りたたみ対応

## ツールの思想

- 本ツールは「公開可否判断のための最低限QA」を目的とする
- 網羅的な品質保証チェックリストではない
- "正解を断定しない"。判断材料を提供する
- UIはシンプル（1ページで完結）

## 技術スタック

- Vite
- React 18
- TypeScript
- CSS（モダンなスタイリング、SP対応、ダークモード対応）

## 使い方

1. 画面上部の条件選択エリアから以下を選択：
   - **技術スタック**：HubSpot / WordPress / 静的サイト / その他CMS
   - **制作種別**：新規制作 / リニューアル・リプレイス

2. 条件を選択すると、該当するチェックリストが表示されます

3. 各チェック項目をクリックしてチェック状態を変更できます

4. セクションヘッダーをクリックして折りたたみ/展開できます

5. 「チェックリストをコピー」ボタンをクリックすると、Markdown形式でクリップボードにコピーされます

## 機能

- 条件に応じたチェックリスト表示
- チェックボックスでの状態管理
- Markdown形式でのコピー機能
- セクション折りたたみ機能
- SP対応レスポンシブデザイン
- ダークモード対応

## チェックリストの構成

### 公開前QA（常に表示：コア）
- 表示・可読性
- 導線・リンク
- 公開状態
- 成果点（存在する場合）
- 最低限の計測（目的がある場合）

### リニューアル時のみ
- リダイレクト確認
- noindex設定確認
- Search Console対応

### 技術スタック別追加QA
- 静的サイト
- WordPress
- HubSpot
- その他CMS（汎用）

### 公開後QA（参考：折りたたみ推奨）
- 計測確認
- 広告流入後の挙動
- Search Consoleエラー
- リダイレクト実測
- キャッシュ・CDN影響
